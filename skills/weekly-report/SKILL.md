# AI 编程周报生成系统

## 概述

本 Skill 实现「AI 编程领域」周报的端到端生成：从多源数据采集，经语义分析、去重、评分，筛选 Top 15 条目供人工确认，最终一次性产出三种分发格式。

**核心定位**：聚焦「AI 编程」赛道 — AI IDE（Cursor / Windsurf / claude code、open code 、codex 等）、AI 编程 Agent、Coding LLM、MCP、Prompt 工程、Vibe Coding、AI 辅助开发最佳实践。

**产出目录**：所有输出文件存放至项目根目录下 `output/weekly-report/`，命名格式 `[类型]_[YYYYMMDD-HHmm].[ext]`。

---

## 执行流程

### Phase 0：环境准备

1. 确认产出目录存在，不存在则创建：`output/weekly-report/`
2. 记录当前日期时间，用于文件命名（格式 `YYYYMMDD-HHmm`）
3. 确定本次周报覆盖的时间范围（默认过去 7 天）

### Phase 1：数据采集

从两类来源并行采集：

#### 1A. 自动采集（必选）

读取 `references/sources.md` 获取完整数据源清单，使用 WebSearch 逐一扫描以下维度：

- **模型发布**：新模型、版本更新、基准测试结果
- **工具更新**：AI IDE、编程插件、Agent 框架的新版本 / 新功能
- **开源项目**：GitHub 热门 AI 编程相关项目、重要 Release
- **论文 & 研究**：与代码生成、AI 辅助开发直接相关的论文
- **社区动态**：重要讨论、最佳实践分享、行业趋势

**采集策略**：

```
对 sources.md 中每个源，执行 WebSearch：
  搜索词 = "[源名称] AI coding / AI programming [本周日期范围]"
  补充搜索 = "[源名称] release / update / announcement [本周日期范围]"
```

每条采集结果记录为结构化条目：

```
{
  "title": "标题（中文，英文原标题附后）",
  "source": "来源名称",
  "url": "原始链接",
  "date": "YYYY-MM-DD",
  "category": "分类标签",
  "summary_en": "英文原始摘要（2-3句）",
  "summary_zh": "中文翻译摘要（2-3句）"
}
```

#### 1B. 人工补充（可选）

检查项目中是否存在 `manual-input.md` 文件（搜索顺序：项目根目录 → `output/weekly-report/` 目录）。

- **文件存在且有内容**：解析其中的条目，合并到采集池
- **文件不存在或为空**：静默跳过，不报错

`manual-input.md` 的格式模板参见 `assets/manual-input-template.md`。

### Phase 2：内容分析

对采集池中所有条目执行以下处理：

#### 2A. 分类

为每条内容打上分类标签（可多标签）：

| 分类        | 说明                                        |
| ----------- | ------------------------------------------- |
| IDE         | Cursor / Windsurf / VS Code AI 插件等       |
| CLI         | Claude Code / Aider / Continue 等命令行工具 |
| Agent       | AI 编程 Agent、自主编码系统                 |
| MCP         | Model Context Protocol 相关                 |
| LLM         | 大模型发布、能力评测、API 更新              |
| Framework   | LangChain / CrewAI / AutoGen 等框架         |
| Prompt      | Prompt 工程、提示词技巧                     |
| Practice    | AI 编程最佳实践、工程化方法论               |
| VibeCoding  | Vibe Coding 相关讨论与案例                  |
| Paper       | 学术论文                                    |
| Open Source | 开源项目                                    |
| Industry    | 行业趋势、融资、战略动态                    |

#### 2B. 去重

基于语义相似度去重：

- 同一事件的不同来源报道 → 合并为一条，保留最权威来源
- 标题/摘要相似度 > 80% → 标记为重复，人工确认时提示

#### 2C. 评分

对每条内容按 5 个维度打分（每项 1-10 分），计算加权总分：

| 维度   | 权重 | 说明                            |
| ------ | ---- | ------------------------------- |
| 相关性 | 30%  | 与「AI 编程」赛道的直接相关程度 |
| 新颖性 | 25%  | 是否为全新发布/突破性进展       |
| 实用性 | 25%  | 对程序员日常工作的实际影响      |
| 影响力 | 15%  | 行业影响范围和深度              |
| 时效性 | 5%   | 发布时间的新鲜度                |

**总分计算**：`score = 相关性×0.3 + 新颖性×0.25 + 实用性×0.25 + 影响力×0.15 + 时效性×0.05`

### Phase 3：筛选与确认

1. 按总分降序排列所有条目
2. 取 Top 15 条，生成候选列表
3. 输出候选列表供人工确认，格式：

```markdown
# 本周 AI 编程周报候选（共 N 条，已选 Top 15）

| #   | 分数 | 分类 | 标题 | 来源 | 一句话摘要 |
| --- | ---- | ---- | ---- | ---- | ---------- |
| 1   | 9.2  | IDE  | ...  | ...  | ...        |
| ... |

请确认：
- 需要删除哪些条目？（输入编号）
- 需要调整排序吗？
- 有需要补充的内容吗？
```

1. 等待用户确认后进入 Phase 4
2. 如用户直接说"确认"或"没问题"，使用当前列表继续

### Phase 4：三格式输出

基于确认后的条目列表，一次性生成三种格式。每种格式的模板和写作要求参见 `references/output-templates.md`。

#### 4A. 周报精选稿

- 文件名：`weekly_[YYYYMMDD-HHmm].md`
- 结构化信息汇总，便于快速浏览
- 包含：本周概览、分类条目、每条的标题 + 摘要 + 链接 + 点评

#### 4B. 公众号深度改写稿

- 文件名：`article_[YYYYMMDD-HHmm].md`
- 基于周报内容深度改写，遵循「程序员 AI 破局指南」人设风格
- 约 2000 字，有观点、有深度、有节奏
- **必须执行去 AI 味检查**（参照 content-creator 技能的 12 项检查清单）

#### 4C. 视频脚本拆写稿

- 文件名：`script_[YYYYMMDD-HHmm].md`
- 从周报中挑选 3-5 个最有话题性的条目
- 每条拆写为 60-90 秒的口播段落
- 完全口语化，开头必须是 Hook

### Phase 5：存档

1. 将三份文件写入 `output/weekly-report/` 目录
2. 同时生成一份采集日志 `collect-log_[YYYYMMDD-HHmm].json`，记录：
   - 采集时间、覆盖范围
   - 原始采集条目数、去重后数量、最终入选数量
   - 每条的完整评分明细 + **URL 字段（必填）**

   每条 item 必须包含以下字段：
   ```json
   {
     "rank": 1,
     "title": "标题",
     "score": 9.65,
     "category": "分类标签",
     "date": "YYYY-MM-DD",
     "url": "主要原始链接（来自 WebSearch 结果，必填）",
     "url_secondary": "备用/补充链接（可选，来自同一事件的第二来源）",
     "relevance": 10,
     "novelty": 10,
     "utility": 9,
     "impact": 10,
     "timeliness": 8
   }
   ```

   **URL 采集规则**：
   - `url` 必须是 WebSearch 返回结果中的真实链接，不得编造
   - 优先选官方来源（官方博客、Changelog、GitHub Release）
   - `url_secondary` 优先选第三方权威报道（TechCrunch、InfoQ、VentureBeat 等）
   - 如果某条目找不到有效 URL，`url` 填 `"N/A"` 并在 `url_note` 字段说明原因
3. 输出完成摘要，列出所有生成的文件路径

## 注意事项

- **聚焦 AI 编程**：严格过滤与 AI 编程无关的通用 AI 新闻（如纯学术 AGI 讨论、AI 绘画、AI 音乐等），除非它们对编程工作流有直接影响
- **中英双语**：原始英文内容保留，同时提供准确的中文翻译，翻译要自然通顺，技术术语保留英文
- **人工确认环节**：Phase 3 必须暂停等待用户确认，这是质量保障的关键节点
- **幂等性**：每次执行产生独立的新文件，不覆盖历史产出
- **内容风格**：公众号深度改写遵循人设（康健 / 程序员 AI 破局指南），坚定、专业、去 AI 味、反爹味
