# AI 编程内容角度模式库

> 从「程序员破局指南」已有内容中提炼的高共鸣角度范式。拿到一条动态/痛点后，先对着这张表找最合适的切入口。

---

## 一、认知反转型（最高传播力）

**本质**：读者以为是 A，实际上是 B。打破惯性认知，制造「哦原来如此」。

| 角度模板 | 示例 |
|---------|------|
| 「大家以为是 X 的问题，其实是 Y 的问题」 | 「以为 git push 卡是网络问题，其实是历史大文件」 |
| 「你以为在学工具，其实在培养判断力」 | 「用 Claude Code 不是在学命令，是在学如何拆解问题」 |
| 「速度慢不是因为你不努力，而是因为你在用错误的方式努力」 | 「AI 编程提效慢，不是 prompt 写得不好，是没有 Skill」 |

**触发条件**：输入里有反常识的事实、错误的初步假设被推翻、大家普遍误解的概念。

---

## 二、痛点具象化型（最高共鸣）

**本质**：把程序员心里有但嘴上说不清的感受，用语言精准钉住。

### 核心痛点类别

**替代焦虑**
- 「会被 AI 替代吗」→ 进一步挖：「什么样的程序员才不会被替代」
- 「写了 10 年代码，AI 3 秒就能写出来」

**效率瓶颈**
- 反复跟 AI 解释上下文，比自己写还慢
- AI 给的代码不能直接用，改起来比写还累
- 不知道该问 AI 什么，脑子里一片空白

**学习焦虑**
- Claude Code / Cursor / Codex 每天都在更新，跟不上
- 看了很多教程，不知道怎么在项目里真正用起来
- 团队里其他人用 AI 越来越快，自己还在用老方法

**职业定位**
- 工具会了，但不知道能不能涨薪
- 公司要求用 AI，但没有人教怎么用

**触发条件**：输入是一个问题或痛点描述，要把「模糊感受」转成「精准痛点 + 破局路径」。

---

## 三、方法论提炼型（最高收藏）

**本质**：把一个具体的解法，提炼成可迁移的思维框架。

| 角度模板 | 示例 |
|---------|------|
| 「解决 X 问题的通用套路」 | 「排查 AI 编程卡点的 3 步法：现象→线索→根因」 |
| 「从 X 案例总结出的 Y 原则」 | 「从 git 大文件案例总结：预防胜于治疗的 3 个检查点」 |
| 「N 个让 AI 协作效率翻倍的操作」 | 「5 个让 Claude Code 更懂你的上下文技巧」 |

**触发条件**：输入是一段实战经历、工具使用心得、或工作流改进，要把经验从「单次」升维到「可复用」。

---

## 四、工具对比评测型（最高搜索量）

**本质**：帮读者在选择上做决策，减少试错成本。

| 角度模板 | 示例 |
|---------|------|
| 「X vs Y，我用了 N 个月后的真实感受」 | 「Cursor vs Claude Code，用了 3 个月后的真实感受」 |
| 「什么情况下用 X，什么情况下用 Y」 | 「什么项目适合 Cursor，什么项目适合 Claude Code」 |
| 「X 的这 3 个功能，99% 的人没用上」 | 「Cursor 的这 3 个功能，99% 的人没用上」 |

**触发条件**：输入是工具更新、新功能发布、或两个工具的差异。

---

## 五、实战案例记录型（最高信任感）

**本质**：「真实的人遇到了真实的问题，用 AI 解决了」。有名有姓有截图，读者相信。

结构锚点：
- 问题现象（越具体越好，有报错信息、数据更好）
- 初步判断是什么 vs 实际根因是什么（认知转变是核心）
- 解决步骤（带命令/截图，可复现）
- 下次遇到的人可以直接用的清单

**触发条件**：输入是一段实际解决问题的过程记录。→ 直接推荐 `coding-session-to-tutorial` 技能。

---

## 六、趋势解读型（最高时效）

**本质**：新动态发布了，帮程序员翻译：这对我意味着什么？我需要做什么？

| 角度模板 | 示例 |
|---------|------|
| 「X 发布了，普通程序员需要知道的 3 件事」 | 「Claude Code 更新了，普通程序员需要知道的 3 件事」 |
| 「X 的出现，会改变哪类程序员的工作方式」 | 「Codex 的出现，会改变哪类程序员的工作方式」 |
| 「从 X 的发布，我看到了 Y 的信号」 | 「从 Claude 4 的发布，我看到了 Agent 时代的信号」 |

**触发条件**：输入是新工具发布、重大更新、行业动态。

---

## 七、「我踩过的坑」型（最高情感连接）

**本质**：坦诚暴露自己的错误和代价，让读者省掉这段路。

| 角度模板 | 示例 |
|---------|------|
| 「我在 X 上浪费了 N 个月，才发现真正重要的是 Y」 | 「我在优化 prompt 上浪费了 3 个月，才发现 Skill 才是正解」 |
| 「刚开始用 X 时，我犯了这 3 个低级错误」 | 「刚开始用 Claude Code 时，我犯了这 3 个低级错误」 |

**触发条件**：输入里有教训、弯路、认知升级的经历。
