# 选题评估标准

> 拿到一个角度后，用这张表决定「要不要做、怎么做」。

---

## 五问评估框架

每个候选选题过一遍，通过 3 个以上立项，通过 2 个酌情，1 个以下放弃。

```
[ ] 1. 符合定位吗？
       → 核心领域：Claude Code / Cursor / Codex / AI 编程方法论 / 程序员职业破局
       → 读者画像：有 1-5 年经验的程序员，想用 AI 提效但还没跑通

[ ] 2. 程序员真实需要吗？
       → 有具体痛点，不是我觉得他们需要
       → 标准：读者看到标题会想「这说的就是我」

[ ] 3. 我有独特视角吗？
       → 有真实实践经历 / 亲自踩过的坑 / 自己的判断
       → 不是整合公开资料，是「别人讲不了这个角度」

[ ] 4. 可以多渠道转化吗？
       → 长文 → 短视频脚本（工作流 D）
       → 长文 → 小红书（工作流 F）

[ ] 5. 能与已有内容连接吗？
       → 能引用之前的文章，或为之后的文章铺垫
       → 不是孤立的话题
```

## 内容类型与推荐工作流映射

| 输入类型 | 最匹配的角度模式 | 推荐工作流 | 理由 |
|---------|---------------|----------|------|
| 一条 AI 工具新闻/更新 | 趋势解读型 / 工具对比型 | 工作流 B（灵感孵化） | 需要补充背景和个人判断，从话题展开 |
| 一段实战问题解决记录 | 实战案例型 / 方法论提炼型 | 先跑 `coding-session-to-tutorial` → 再用工作流 C 润色 | 结构化整理后再做风格加工 |
| 一个程序员痛点/困境 | 痛点具象化型 / 认知反转型 | 工作流 B（灵感孵化） | 围绕痛点扩展论证和案例 |
| 已有文章/笔记 | 方法论提炼型 / 认知反转型 | 工作流 C（文章改编） | 在已有基础上提升深度和传播力 |
| 一段录音/语音转录 | 任意 | 工作流 A（录音转文章） | 先整理再判断角度 |

## 角度质量判断

一个好角度满足：**读者看完会做一件事**——转发、收藏、或改变某个行为。

**弱角度信号**：
- 标题是介绍型（「Claude Code 功能介绍」）
- 没有「为什么要关心我」的钩子
- 信息量大但没有判断（「关于 AI 编程的 10 件事」）

**强角度信号**：
- 有反常识（「大家都在学 prompt，但其实……」）
- 有情绪共鸣（「用了 3 个月还是觉得没提效，不是你的问题」）
- 有具体行动（「今天可以做的 1 件事」）

---

## 角度→标题快速推导

确定角度后，用这个公式快速出 3 个候选标题：

```
痛点型：  [读者最怕的事] + [转折/解法]
          例：「用 AI 编程用了半年，效率还不如以前」这才是真正的原因

反常识型：「大家以为 [错误认知]，其实 [真相]」
          例：大家以为 AI 编程就是写 prompt，其实真正的门槛在这里

方法型：  [N] 个 [动作] + [收益]
          例：3 个让 Claude Code 秒懂你需求的上下文技巧

案例型：  [真实场景] + [结果数据/对比]
          例：一个 git push 卡住问题，让我彻底搞懂了 AI 编程的排查思路
```
