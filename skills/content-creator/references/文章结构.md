# 内容创作结构模板

本模板提供四种文章结构模式，旨在生成**2000字以上、逻辑严密、有核心洞见**的专业文章。应根据**素材类型**和**表达目的**灵活选择。

---

## 核心原则：洞见优先 + 人味优先

- **洞见优先**：每篇文章必须有一个让人「哦！原来如此」的核心洞见。不是汇总大众观点，是穿透本质的那一刀。文章写完，你要能用一句话说清楚「读者看完这篇文章，会得到什么以前不知道的认知」。
- **论证完整**：论点 + 案例 + 原理 + 反例 + 结论 = 一个完整的论证闭环。一个观点用「我觉得」支撑，跟没说一样。
- **节奏设计**：好文章有呼吸感。开场高能，中段有转折低谷，结尾升华回味。平铺直叙是大忌。
- **人味优先**：真实 > 完美。具体细节、口语化表达、真实情绪，永远优于精致的模板。参考 `去AI味指南.md`。

---

## 模式一：认知重塑型

**适用场景**：纠正误区、通过现象看本质、提出反直觉观点。
**核心逻辑**：打破旧认知 → 深度剖析谬误 → 建立新框架 → 演绎新认知。
**洞见要求**：核心洞见必须是「反直觉的」——读者看到之前，会认为你的结论是「错的」。

1. **场景切入（约200字）**
   - 还原一个具体场景，让读者觉得「这就是我」。
   - 不要直接给结论，先让读者在自己的认知里待一会儿。

2. **共识与缺陷（约400字）**
   - 承认大众普遍的做法/观点，甚至表示「我也这么认为过」。
   - 然后指出这个共识的逻辑漏洞。
   - 深度要求：不要停在表面，挖掘背后的人性弱点、认知偏差或路径依赖。

3. **核心观点抛出**
   - 一句话说清楚反直觉的结论。
   - 这句话要让读者停下来，重读一遍。

4. **深度拆解（约800字）**
   - **维度一**：从机制/原理层解释「为什么会这样」（可以引入信息论、系统论、行为经济学视角，但要口语化翻译）。
   - **维度二**：对比分析，高手 vs 普通人，短期视角 vs 长期视角。
   - **维度三**：主动预判并回应读者最可能的反驳（「有人会说……但……」）。

5. **实操落地（约400字）**
   - 基于新认知的方法论。
   - 给出具体的 Action List，要有颗粒度（不是「多思考」，是「每次开始任务前，先花5分钟写下……」）。

6. **升华（约200字）**
   - 从技术层面上升到思维/职业维度，留下余味。
   - 不要用「写在最后」，用一个具体的画面或场景收尾。

## 模式二：实战复盘型

**适用场景**：分享个人经历、项目复盘、踩坑故事。
**核心逻辑**：遭遇挑战 → 探索弯路 → 顿悟破局 → 系统化抽象。
**洞见要求**：复盘不是流水账，要抽象出一个「通用规律」——别人在类似处境里也能用的结论。

1. **Hook（约200字）**
   - 用一个糟糕的结果或惊人的数据开场，不要用「今天我来分享」这种废话开头。
   - 交代背景复杂度，让读者感受到挑战的真实重量。

2. **冲突与困境（约400字）**
   - 遇到了什么反常识的困难？
   - 尝试了哪些常规方法失败了？要详细描述失败过程（「我试了三种方案，全挂了」），而不是一笔带过。
   - 这里要有真实的挫败感和情绪，读者要能感受到当时的窒息。

3. **转折与顿悟（约300字）**
   - 发现破局点的关键时刻。
   - 深度要求：描述思考过程的转变，而非仅展示结果（「我突然意识到，问题根本不在……」）。

4. **解决方案演进（约600字）**
   - **V1.0**：初步解决，但有瑕疵——写出来，展示真实的探索轨迹。
   - **V2.0**：迭代优化，解决深层问题。
   - **最终方案**：展示架构图、代码片段或对比数据（有具体数字才有说服力）。

5. **抽象与方法论（约300字）**
   - 把个人经验总结为通用的 SOP 或原则（这是复盘的核心价值）。
   - 要说清楚「适用边界」——这个方法在什么情况下有效，在什么情况下不适用。

6. **结语（约200字）**
   - 对行业或个人成长的真实反思，不要套路升华。
   - 用一个画面或疑问收尾，给读者留下思考空间。

## 模式三：方法论/指南型

**适用场景**：讲解具体技能、工具使用、学习路线。
**核心逻辑**：痛点 → 道（心法） → 法（框架） → 术（步骤） → 器（工具）。
**洞见要求**：方法论本身就是洞见。你的框架要有「命名」和「边界」，让读者觉得「以前从来没这么想过，但说的太对了」。

1. **痛点唤醒（约200字）**
   - 不讲大道理，直接戳「为什么你学不会/做不好」。
   - 列举 2-3 个具体的错误姿势（要有画面感，读者要认出自己）。

2. **心法层（约400字）**
   - 在讲操作前，先讲心法。
   - 回答「为什么要学这个？它的底层价值是什么？」不回答这个问题，用户学完也不会用。
   - 这里可以引入一个「反直觉观点」，打破读者的固有认知。

3. **框架层（约400字）**
   - 给出一个好记的模型（「三层架构」「四步法」「2×2矩阵」）。
   - 详细解释每个模块的定义和边界——「什么情况算A，什么情况算B」。
   - 框架要有「命名」，好记的名字比好的内容传播得更远。

4. **执行层（约800字）**
   - Step-by-Step 指南：每一步都要有「Why」（为什么这样做）和「How」（具体怎么做）。
   - 避坑指南：这一步新手最容易犯什么错？后果是什么？（用真实案例说明，不要用假设）
   - 最佳实践：高手是怎么做的？和普通人的做法差在哪里？

5. **行动号召（约200字）**
   - 给出一个马上能做的最小行动（今天就能做，5分钟内能上手）。
   - 降低门槛，减少读者的启动阻力。

## 模式四：洞见爆破型 ⭐️ 新增

**适用场景**：强观点文章、行业批判、重新定义某个概念、反驳流行观念。
**核心逻辑**：开场即颠覆 → 建立信任 → 多维论证 → 预设反驳 → 更大框架升华。
**特别说明**：这是最难写、天花板也最高的模式。文章的核心观点必须足够「大胆」，让读者第一眼觉得「这人说的也太狠了」，读完觉得「他说得对」。

1. **开场颠覆（约200字）**
   - 第一段就抛出核心观点，不要铺垫太久。
   - 观点必须够犀利、够反常识——「大家都说X是对的，我认为这件事根本上是错的」。
   - 用一个具体场景或数据支撑第一拳。

2. **建立信任（约300字）**
   - 不能让读者觉得这只是哗众取宠。
   - 主动承认这个观点的「争议性」：「我知道很多人不同意，但听我说完……」
   - 展示自己的「资格」——为什么你有资格说这个？（亲历的案例，或深度的观察）

3. **多维论证（约800字）**
   - **论证一**：从事实层面（数据、案例、观察）证明现有认知的局限。
   - **论证二**：从机制层面（原理、模型）解释为什么会这样。
   - **论证三**：从历史/类比层面，提供参照系（「类似的事在……领域发生过」）。

4. **预设反驳（约300字）**
   - 主动提出最强的反对意见：「最有力的反驳是……」
   - 不要回避，直接正面回应，化解掉。
   - 回应不是「对方说的也有道理」，是「这个反驳成立的前提是X，但X本身就有问题」。

5. **边界厘清（约200字）**
   - 好的观点要有边界：这个结论在什么情况下成立，在什么情况下不成立。
   - 没有边界的观点，看起来很爽，实际上站不住脚。

6. **更大框架（约200字）**
   - 把这个具体观点放入一个更大的框架里，提升思维层次。
   - 让读者感觉「这不只是在聊一件事，这是在聊一种思维方式」。

## 如何选择结构模式？

| 输入素材是… | 核心目的是… | 推荐模式 |
| :--- | :--- | :--- |
| 反直觉的想法/对现状的吐槽 | 改变读者认知 | **模式一：认知重塑型** |
| 具体项目经历/踩坑记录 | 建立信任/证明能力 | **模式二：实战复盘型** |
| 好用的工具/学习方法 | 交付实用价值 | **模式三：方法论/指南型** |
| 强烈的观点/行业批判 | 引发讨论/树立权威 | **模式四：洞见爆破型** |

## 文章节奏设计

**好文章的能量曲线**，不是一条平线，而是有起伏的：

```
能量
  │
高 │  ██        ████
  │ ████      ██████
  │ █████   ████████
  │ ██████████████████
低 │___________________________→ 进度
    开场  中段  转折  高潮  收尾
```

- **开场**：必须高能。第一段决定读者是否继续读，不能用废话暖场。
- **中段**：可以有一个「低谷」——展示困境、承认局限、引入反例。低谷让后面的突破更有力量。
- **转折**：文章至少有一个「但是」的转折点，打破读者预期。
- **高潮**：文章最核心的洞见/最精彩的论证，要放在读者刚好「入戏」的位置（通常是中后段）。
- **收尾**：不要用总结收尾，用一个具体的画面/疑问/行动收尾，留下余味。

## 通用深度组件

在适当位置插入以下组件，提升文章专业度和深度：

**原理卡片**：用引用块解释涉及的底层原理（如：认知偏差、系统思维、边际效应），**但必须口语化翻译**，不能直接堆术语。

**反直觉对比**：「大众认知 vs 真实情况」对比，是让读者停下来的利器。

**边界说明**：明确这个观点在什么条件下成立，在什么条件下不成立。没有边界的观点，容易被一个反例打倒。

**权威背书**：引用经典书籍、技术文档、实验数据来佐证观点——但不要只堆引用，要说明「这对我们意味着什么」。

**读者预判**：在读者可能产生疑问的地方，主动说「有人会问……」然后直接回答。这会让读者觉得作者在跟他对话，而不是单向输出。
