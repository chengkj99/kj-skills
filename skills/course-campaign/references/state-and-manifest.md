# 进度文件与章节清单规范

course-campaign 靠两样东西工作：**章节清单**（要写什么）和**进度文件**（写到哪了）。本文定义它们的格式。

---

## 一、章节清单（Manifest）

清单是「这门课要写哪些章、每章用什么材料」的总表。可以由用户直接贴在对话里，也可以放成一个文件让 campaign 读。

### 每条章节的字段

| 字段 | 必需 | 说明 |
|------|------|------|
| 编号 ID | ✅ | 如 `CC-005`，用于排序和续写定位 |
| 标题 | ✅ | 如「CLAUDE.md 放哪、写什么、/init 怎么用」 |
| 参考材料路径 | 建议 | 一个或多个路径 / glob，直接透传给 course-generator |
| 附加要求 | 可选 | 本章特有的要求（如「重点讲 X」「对比 Y」） |

### 推荐写法（表格）

```markdown
| # | ID | 标题 | 参考材料 | 附加要求 |
|---|-----|------|---------|---------|
| 1 | CC-005 | CLAUDE.md 放哪、写什么、/init 怎么用 | <wiki>/raw/papers/指南.md §3；<wiki>/raw/papers/claude-code-101/ | |
| 2 | CC-006 | Skills 目录结构与触发逻辑 | 指南 §3；<wiki>/raw/papers/learn-claude-code/07-*.md | |
```

`<wiki>` 这类占位根路径在清单开头声明一次，campaign 透传给 course-generator 时展开成绝对路径。

### 区间简写

用户给 `CC-005..CC-014` 这类区间时，campaign 需要拿到对应的完整清单（标题 + 材料）才能干活。处理顺序：
1. 用户同时给了清单表 → 直接用。
2. 只给区间没给表 → 先把区间内每章的标题和材料路径问清楚 / 让用户贴出来，**不要自己编标题和材料**。

## 二、进度文件（Progress）

- **路径**：`<输出目录>/.course-campaign-progress.md`
- **作用**：唯一的「写到哪了」真相源。每章写完必更新。compact / 中断 / 隔天重来都靠它恢复。
- **只增量改状态行**，不重写历史。

### 状态机

| 状态 | 含义 | 下一步 |
|------|------|--------|
| `pending` | 还没写 | 轮到它就写 |
| `done` | 已交付落盘 | 跳过，不重写 |
| `blocked` | 材料不足等原因卡住 | 暂停报告用户，不跳过往后写 |

**续写规则**：从上往下找**第一个非 `done`** 的章节开始。遇到 `blocked` 停下报告，不要绕过它写后面的（会断衔接）。

### 文件模板

```markdown
# course-campaign 进度：<课程名>

- 输出目录：<输出目录>
- 风格基线：<如 CC-001～CC-004>
- 里程碑：<如 CC-007 / CC-010 / CC-013 写完后 compact；或「默认每 3 章」>
- 更新时间：<YYYY-MM-DD>

| ID | 标题 | 状态 | 输出文件 | 类型 | 字数 | 备注 |
|----|------|------|---------|------|------|------|
| CC-005 | CLAUDE.md 放哪… | done | lessons/.../CC-005-CLAUDE-md.md | 工具操作型 | ~2100 | |
| CC-006 | Skills 目录结构… | pending | | | | |
| CC-007 | Plan Mode… | pending | | | | |
| …    | … | pending | | | | |
```

### 初始化

进度文件不存在时，按清单生成它：所有章节状态填 `pending`，表头填课程名 / 输出目录 / 风格基线 / 里程碑。然后从第一章开始。

### 每章写完后的更新

- 状态 `pending` → `done`
- 补齐：输出文件路径、章节类型（course-generator 报告里有）、字数
- 刷新「更新时间」

## 三、衔接信息怎么存

上一章「引出下一章的方式」不需要单独存进度文件——它是**临时上下文**，每章 Step A 现读现用即可（读上一章结尾两段）。进度文件只存「写到哪了 + 落盘事实」，不存正文衔接细节，保持轻量。
