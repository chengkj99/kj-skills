# paid-content-review：付费内容发布前质检

把一篇或一整门付费内容，在上线前过一遍系统化质检，产出**按严重程度排序、点名到具体位置、可直接执行**的审查报告。

核心判断不是「单章写得好不好」，而是**这套内容是否真的值得付费购买**——整门课要构成一套系统课程而非 N 篇松散文章；单篇要拉开与免费科普的差距。

**默认发布渠道：知识星球基础内容。** 学员是按篇付费注意力的——每一篇都要独立站得住，不能靠「整课体系」掩盖单篇注水。

付费内容的风险往往不在单章质量，而在**全局一致性 + 付费价值 + 篇级用户体验**：编号断层、跨章重复、深浅不均、跟不上工具版本、太像免费文章、术语绕、结构散。向 AI 泛问「有什么问题」只能得到泛泛框架，必须先把**审什么维度 + 审多大范围 + 用什么标准**三件事固定下来。

## 适用边界（先确认是不是该用我）

- ✅ 用我：内容已写完，要在**收费 / 公开发布前**做把关——找硬伤、评估付费价值、判断能不能现在上线。
- ❌ 不用我：还在写正文 → `course-generator`；还没想好写什么 → `ai-programming-topic-planner`；把一段实战记录整理成案例 → `coding-session-to-tutorial`。

## 知识星球篇级必过：用户视角三硬指标

课程将作为**知识星球基础内容**发布。审查时（单篇全文 / 整课深读章）必须从**付费学员读完这一篇**的视角，逐篇核对以下三条——**缺一条即记问题**，严重缺深度或结构混乱可升为 P0/P1：

| # | 硬指标 | 用户会怎么感受 | 必须看到什么 | 失败即记 |
|---|--------|----------------|--------------|----------|
| **U1** | **有深度、有价值** | 「不是表面介绍，带走能用的东西」 | 至少一项可复用增量：方法/步骤、判断标准（何时用/何时别用）、踩坑点、取舍对比、可复用模板 | 只讲「是什么」；无作者增量；通篇像免费科普/官方文档摘抄 |
| **U2** | **专业又易懂** | 「术语靠谱，但讲得不绕；新手跟得上，老手也有收获」 | 术语准确；先给直观解释再给术语（或术语后立即白话）；关键处有进阶注脚/对比/边界，让老手也能得点 | 术语堆砌难懂；或过度口语丢专业度；只有入门没有进阶增量；只有高深没有新手入口 |
| **U3** | **逻辑清晰** | 「读起来顺，知道为什么学、学什么、怎么做、下一步」 | 叙事大致遵循：**开篇问题意识 → 概念/方法 → 示例/操作 → 小结/下一步**；段落不跳戏、小节标题能串成故事线 | 开篇无问题/场景；概念与操作倒置；有概念无示例；缺小结或下一步；中途大段跑题 |

三条与维度的关系：

- U1 ↔ 主要落在 **D6 / D10**（深浅与付费价值），细则见 D14
- U2 ↔ 主要落在 **D9**（表达一致性）+ D14 的易懂/双层读者
- U3 ↔ 主要落在 **D14**（单篇叙事结构）；整课衔接另见 D3

深读时先过三硬指标再评其他维度；报告里用「U1/U2/U3」或「深度/易懂/逻辑」点名，便于作者改稿。

## 第一步：判定审查模式

| 模式 | 何时用 | 报告模板 |
|------|--------|----------|
| **整课模式（course）** | 审查对象是一门课：有总纲/课表/manifest + 多篇正文目录 | `references/course-review-report.md` |
| **单篇模式（single）** | 审查对象是单篇/单讲内容 | `references/single-piece-review.md` |

用户可显式指定模式；未指定时按上表自动判定（给到目录/课表 → 整课；给到单个文件 → 单篇）。

**专项子模式**（可叠加在两种模式上，用户点名时启用，否则做全维度审查）：

- **准确性专项**：只扫命令/参数/配置/API，逐条标注是否符合当前版本（最该单独做的一项，见 D5）。
- **定价视角**：站在「定价 ¥X 的付费学员」角度，专审撑不撑得起价、和免费有没有拉开差距（D10/D11）。
- **单章深审**：在整课里挑一章，只跑「准确性 + 深浅 + 可操作性 + 三硬指标（U1–U3）」，逐行给问题与改法。
- **知识星球篇体验专项**：只审 U1/U2/U3（深度价值 / 专业易懂 / 逻辑结构），适合改稿期快速过篇。

---

## 第二步：先收集三要素（缺则先问，别盲审）

这三件事决定判定线。**缺任意一项就先简短追问，不要凭空假设后开审**——盲审只会得到泛泛结论，这是本技能与「直接丢给 AI 找问题」最大的区别。

1. **目标读者** —— 零基础 / 有经验转过来 / 团队内训？决定深浅与分层标注（直接影响 U2「新手能跟、老手有收获」怎么判）。
2. **定价与对标** —— 多少钱、对标谁（某免费教程 / 某付费课）？决定「付费价值」和「定价支撑」的判定线。未说明时，默认按**知识星球会员基础内容**理解：单篇也要撑得起「值得付费进圈」的预期。
3. **当前阶段** —— 终审找硬伤（输出清单）/ 还在改稿期要建议（可给改法甚至改稿）？决定输出形态。

整课模式再补两项（有就给，没有可跳过）：**销售页/课程承诺**与**毕业项目设计**——用于审「商业吸引力」是否兑现承诺。

可选补一项：**发布渠道**——默认「知识星球基础内容」；若实际是独立专栏/小报童/官网课，说明后相应调整包装维度（D12），但 U1–U3 仍适用。

> **无法即时追问时**（批处理、子代理、headless 等场景）：不要因为缺三要素就停摆。用一组合理的假设值开审，但**必须在报告开头显式标注「以下为假设值，终审前需创作者确认」**，并让结论相应保守。渠道默认假设为「知识星球基础内容」。

## 第三步：按范围读取材料（含抽样策略）

**整课模式**——内容量大，不可能每章都深读，按三层抽样，并在报告里如实说明抽样范围：

1. **全量读总纲/manifest**：拿到模块结构、章节清单、status、编号、备注。这是后续核对的基准。
2. **全量轻扫所有正文文件**：只为核对结构完整性（D4）——文件是否都在、编号是否连续、有无空章/墓碑页、文件名拼写、status 与实际是否相符。轻扫可用文件列表 + 每篇首屏/字数。
3. **专项全量扫描**：命令/参数/配置/API（D5）这类「跑不通就退款」的内容，**必须全量扫**，不能抽样。扫的时候做**横向比对**——同一命令在「概览/速查表」与「动手章」里是否写法一致（实测中最硬的 P0 就是概览表用了不存在的包名、与动手章自相矛盾）。
4. **章首衔接句通扫**：抽取每章开头的「上一章…」承上启下句，核对它引用的前序内容是否与实际编号顺序一致——重构调过模块顺序后，这里最容易留下「引用了还没学到的后置模块」的断层（D3）。这一步成本低、抓断层最准，整课模式必做。
5. **代表性深读**：每个模块挑 1–2 章代表章深读（开篇章 + 重点章 + 疑似薄弱章），评深浅、可操作性、付费价值，并**强制过 U1/U2/U3 三硬指标**。

**单篇模式**——通读全文，全维度评；开篇先过 U1/U2/U3，再评其余维度。

---

## 第四步：按维度审查（统一维度模型）

每条发现都要**点名到具体位置**（整课：模块/章节号；单篇：行号或小节），并给**为什么是问题 + 怎么改方向**。泛泛而谈（「整体可以更深入」）不算有效发现。

| # | 维度 | 适用 | 一句话查什么 |
|---|------|------|--------------|
| **D1** | 学习路径与梯度 | 整课 | 是否逐层推进、有无内容出现过早/过晚、难度跳变 |
| **D2** | 模块分层与边界 | 整课 | 模块职责是否清晰，有无重叠/边界模糊/主题重复/模块内过散 |
| **D3** | 前后依赖 | 整课 | 是否建立清晰先后关系，哪些章依赖未铺垫、该前移/后移、缺承上启下 |
| **D4** | 结构完整性 | 整课 | 以 manifest 核对实际文件：编号连续、缺章/空章/墓碑页、status 一致、文件名拼写 |
| **D5** | 内容准确性与时效 | 两者 | 命令/参数/配置/API/模型说法/路径/工具名是否仍正确，标出上线前必复核项 |
| **D6** | 深浅与梯度 | 两者 | 该展开处是否一笔带过——付费溢价在「比免费深」；是否有方法/判断标准/踩坑 |
| **D7** | 重复与冗余 | 两者 | 跨章 / 篇内是否重复讲同一概念、字数注水 |
| **D8** | 可操作性与交付物 | 两者 | 有无可复现步骤/代码/截图/清单/练习，学完能否产出明确结果而非只「知道」 |
| **D9** | 一致性 | 两者 | 术语、人称、风格、格式、编号、回链是否统一 |
| **D10** | 付费价值 | 两者 | 每讲是否有超免费科普的价值（判断框架/实操/命令/工作流/踩坑/案例/模板） |
| **D11** | 定价支撑 | 整课 | 内容厚度配不配定价、与免费教程的差异点 |
| **D12** | 商业吸引力与包装 | 两者 | 标题/模块名/顺序/毕业项目撑不撑得起销售承诺，哪些标题不像愿意付费学的表达 |
| **D13** | 发布风险综合 | 两者 | 汇总结构/内容/时效/付费感/完整度/篇级体验风险，给「上线前必须解决」项 |
| **D14** | 知识星球篇级体验（U1–U3） | 两者 | 深度价值 / 专业易懂 / 叙事逻辑——每篇是否同时过关 |

**每个维度的详细判定标准（查什么细项、失败信号、增强方向）见 `references/dimension-rubric.md`——开审前先读它。**

单篇模式跳过 D1–D4 与 D11（这些是全局/跨章维度），其余照评，D7/D9 作「篇内」理解；**D14 必跑**。

## 第五步：输出审查报告

- **整课** → 套用 `references/course-review-report.md` 的模板。
- **单篇** → 套用 `references/single-piece-review.md` 的模板。
- **先出报告，不改文件**。仅当用户处于「改稿期」且**明确要求**改时，才在出完报告后动手改。
- **改过时内容时**：正文直接换成现行正确做法，**不要**留下「已归档 / 已弃用 / 不要再装旧包」等考古说明——那是作者内部信息，对学员是噪音（细则见 D5「改稿纪律」）。
- 不要平均用力：明确优先级，把笔墨集中在 P0/P1。
- 报告中须有一节（或表格列）显式覆盖 **U1/U2/U3**，避免只写笼统「付费感不足」。

### 严重程度定义（贯穿所有表格）

- **P0 必改**：上线即翻车——跑不通的命令、缺章/墓碑页、结构断层、与定价严重不符、整篇无深度（纯科普充数）、叙事严重混乱读不下去。不解决不能发。
- **P1 建议改**：明显影响体验或付费感——深浅不均、跨章重复、标题不抓人、缺交付物、术语过绕或过浅、缺小结/下一步、老手读完无增量。发之前尽量改。
- **P2 可优化**：锦上添花——措辞、格式统一、可加的对比表/案例、进阶注脚可更清晰。可上线后迭代。

## 质量自检（交报告前过一遍）

- [ ] 是否点名了**具体模块/章节/行号**，而非泛泛而谈？
- [ ] 每条发现是否给了**为什么是问题 + 怎么改**？
- [ ] 是否**按 P0/P1/P2 排序**、明确了优先级，没有平均用力？
- [ ] 三要素（读者/定价/阶段）是否已确认或已向用户追问？渠道默认是否按知识星球处理？
- [ ] 深读篇/单篇是否逐篇过了 **U1 深度价值 / U2 专业易懂 / U3 逻辑清晰**？
- [ ] 整课模式：是否如实说明了**抽样范围**（哪些深读、哪些只轻扫）？
- [ ] 命令/配置类（D5）是否**全量扫**而非抽样，并标了「上线前必复核」？
- [ ] 若已改稿：过时命令/包名是否已**直接替换为现行写法**，正文里没有「已弃用/已归档/勿用旧包」类读者噪音？
- [ ] 是否**未擅自改文件**（除非用户在改稿期明确要求）？
- [ ] 该合并/删除/前移/后移的章节，是否**直接说了**而不是含糊带过？
