# 整课审查报告模板（course mode）

用于整门付费课程的全局体检。**严格使用以下结构输出**，表格列不要增删。每条发现点名到具体模块/章节号，给「为什么是问题 + 怎么改」。按 P0/P1/P2 排序，不要平均用力。

报告开头用一行交代**抽样范围**（哪些章深读、哪些只轻扫结构、命令是否全量扫），让读者知道结论的覆盖面。

## 维度到报告章节的映射

写报告时，把第四步各维度的发现填到对应小节，不要漏维度也不要重复填：

| 维度 | 落到报告的 |
|------|-----------|
| D1 学习路径、D2 模块分层、D3 前后依赖、D4 结构完整性 | 二、结构问题 + 三、重复断层错位 |
| D7 重复冗余 | 三、重复断层错位（类型=重复） |
| D6 深浅、D10 付费价值、D11 定价支撑 | 四、付费感不足 |
| D8 可操作性与交付物 | 五、需补强（补练习/补交付物/补命令）|
| D12 商业吸引力与包装 | 六、标题与包装 |
| D5 内容准确性与时效 | 七、时效与必复核清单 |
| D9 一致性 | 视情况进二（格式/编号）或四（风格拼接感）|
| D13 发布风险综合 | 八、整改优先级 + 九、最终结论 |

## 写作要求

- **点名到位**：每条都带模块/章节号（如 M4-05），不写「某些章节」。
- **分轻重**：P0 集中在「跑不通 / 缺章 / 结构断层 / 严重不配价」；不要把措辞问题塞进 P0。
- **敢下结论**：该合并/删除/前移/后移的，直接说，并说清理由。
- **内容正确但不该收费**：若某章正确却不值得作为付费一讲，在第四节明确指出并建议降级/合并/删。
- **抽样诚实**：没深读的范围，结论要相应保守，并在开头抽样说明里讲清楚。
