# Source Map｜通用信息源地图

调研时至少覆盖以下来源族。不同主题可增删，但不得只依赖单一来源。

## 1. 官方发布

用途：确认事实、产品能力、API、约束、价格、版本、路线。

常见来源：

- 公司官网 News / Blog / Release Notes / Changelog / Docs
- API 文档
- 帮助中心
- 开发者文档
- 标准组织、监管机构、行业协会
- 云服务商和平台文档

重点提取：

- 发布时间/更新时间
- 新能力
- 适用范围
- 限制条件
- 迁移成本
- 对现有产品架构的影响

## 2. 论文与研究

用途：发现新方法、新评价体系、新数据集、新 benchmark。

常见来源：

- arXiv
- OpenReview
- ACL Anthology
- Semantic Scholar
- Papers with Code
- Hugging Face Papers
- 会议官网：NeurIPS、ICML、ICLR、ACL、EMNLP、CHI、KDD、WWW 等

重点提取：

- 解决的问题
- 方法核心
- 实验设置
- benchmark 是否可信
- 是否有代码/数据
- 与产品场景的距离
- 局限

## 3. 开源项目和 demo

用途：判断是否已经有工程可行性、社区热度和复用价值。

常见来源：

- GitHub Trending / Search / Releases / Issues
- Hugging Face Models / Spaces / Datasets
- npm / PyPI / crates / Docker Hub
- 官方 examples / starter kits

重点提取：

- 最近更新时间
- star/fork/watch 增长
- issue 活跃度
- release 频率
- demo 是否可运行
- 文档质量
- 依赖和许可证
- 生产可用性

## 4. 工程社区

用途：发现真实落地问题、失败案例、采用门槛。

常见来源：

- Hacker News
- Reddit
- Stack Overflow
- 技术博客
- 工程复盘
- Discord/Slack 社区
- Conference talks

重点提取：

- 工程争议
- 常见坑
- 性能/成本问题
- 可靠性问题
- 安全问题
- 维护成本

## 5. 行业专家

用途：发现解释框架、趋势判断和“为什么重要”。

常见来源：

- 专家博客
- newsletter
- 播客
- 投资机构报告
- 研究机构报告
- 高质量演讲、主题演讲、访谈、公开课

重点提取：

- 观点是什么
- 证据是什么
- 是否被多个来源验证
- 是否存在利益相关
- 对产品方向的启发

## 6. 高校和研究机构

用途：从一线高校、实验室、研究中心、课程、seminar、colloquium 和公开演讲中发现正在成形的行业最佳实践。

选择原则：

- 按主题选择高相关高校、实验室、研究中心和行业研究机构。
- 用户明确要求“顶尖高校”时，再用最新权威榜单校准名单，并标注榜单时间、学科适用性和过期风险。
- 高校/机构材料默认是方法、框架或早期趋势信号，落到产品建议前应和官方、论文、工程、竞品或市场信号交叉验证。

常见来源：

- 学校官网 News / Events / Research
- 学院、实验室、行业研究中心、数据/工程/管理/政策研究机构
- course/syllabus、lecture notes、seminar、colloquium
- faculty talk、guest lecture、public lecture、video、transcript、slides

重点提取：

- 是否是最新课程、实验室方向、研究项目或公开讲座
- 观点/方法是否来自可核验论文、项目或课程材料
- 是否能转成产品方法、工程实践或评估体系
- 与行业落地的距离
- 是否被工程社区、竞品或市场信号验证

## 7. 行业影响力人物/权威专家

用途：从领域研究者、创业者、工程负责人、产品负责人、标准制定者、行业协会专家和投资研究者的 keynote、lecture、podcast、interview、fireside chat 中捕捉解释框架和趋势判断。

选择原则：

- 按主题临时识别该领域的高可信人物，不使用固定权威名单。
- 优先选择有公开材料、可核验案例、论文/标准/工程实践支撑的人物。
- 明确区分观点、事实和利益相关。

常见来源：

- 行业会议、学术会议、标准组织会议、企业技术大会、TED、专业协会、研究机构、投资机构、播客和访谈。

重点提取：

- 观点是什么，是否有事实或论文支撑
- 演讲时间、场合、原始视频/文字稿
- 适用边界和利益相关
- 是否被后续产品、论文或工程实践验证
- 对目标产品/流程的可迁移启发

注意：

- 演讲/访谈默认是观点或解释框架，证据等级通常为 C。
- 不得把单场演讲当成事实结论；必须与官方、论文、工程、市场或竞品信号交叉验证。

## 8. 竞品和市场

用途：判断趋势是否已经被产品化和用户验证。

常见来源：

- 竞品官网
- Release notes
- Help docs
- Pricing
- Case studies
- Product Hunt
- G2/Capterra
- App Store/Chrome Web Store
- 社媒评论
- 招聘 JD

重点提取：

- 新功能
- 目标用户
- 定价或包装方式
- 用户评价
- 差评和未满足需求
- 竞争壁垒
- 商业化信号

## 9. 内部资料

用途：把外部趋势转成真实产品机会。

用户提供时优先使用：

- 用户访谈
- 客服工单
- 销售反馈
- 埋点数据
- 历史 PRD
- 会议纪要
- 事故复盘
- 知识库文档

重点提取：

- 高频痛点
- 真实使用场景
- 用户语言
- 决策链路
- 现有产品约束
- 可验证指标
