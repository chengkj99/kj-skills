# AI 日报价值评分规则

## 评分维度与权重

`score = 相关性×0.30 + 可执行性×0.25 + 新颖性×0.20 + 影响面×0.20 + 时效性×0.05`

| 维度 | 权重 | 高分特征 |
|------|------|---------|
| **相关性** | 0.30 | 直接影响 AI 编程工作流（IDE / Agent / CLI / MCP / 代码模型） |
| **可执行性** | 0.25 | 7 天内可行动，有仓库/文档/发布说明入口 |
| **新颖性** | 0.20 | 新发布、新能力、新基准结果（非旧闻转述） |
| **影响面** | 0.20 | 头部厂商、高采用率工具，影响范围广 |
| **时效性** | 0.05 | 24h 内发生 |

## 优先级分级

| 级别 | 分数 | 含义 | 处理方式 |
|------|------|------|---------|
| **P0** | ≥ 9.0 | 重大发布/突破 | 放「今日头条」，建议当天行动 |
| **P1** | 8.0–9.0 | 重要更新 | 放「产品&模型」「研究&论文」 |
| **P2** | < 8.0 | 趋势信息 | 放「简讯」，一行说明 |

## 去重规则

- 同一事件多来源：保留官方原始链接，合并为一条
- 文本语义高度重复：只保留评分最高条目
- 同一人同一话题多条：合并要点，保留一条

## 行动建议模板

| 内容类型 | 行动建议方向 |
|---------|------------|
| 模型/API 发布 | 「评估迁移成本」「运行基准对比」「升级依赖版本」 |
| 开源工具/仓库 | 「最小可运行验证（star + clone + quickstart）」 |
| 研究论文 | 「阅读 Abstract + 结论，判断是否影响当前方案选型」 |
| 大 V 观点 | 「提炼核心论点，结合自身项目判断适用性」 |
| 产品功能更新 | 「在当前工作流中找一个具体场景试用」 |
