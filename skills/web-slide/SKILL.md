# web-slide · 网页演讲稿生成

## 概述

将用户的演讲主题/大纲生成为完整的单文件 HTML 演讲稿。
所有交互能力（翻页、备注、代码高亮、流程图、PDF 导出）自动继承自框架，用户只需关注内容。

**框架源文件**：skill 内置 `assets/skills-deck.html`（随 skill 分发，自包含，无需依赖外部项目）

---

## 输入收集

启动后先确认以下信息（用户已提供的直接使用，未提供的询问）：

| 字段 | 说明 | 默认值 |
|------|------|--------|
| 主题 | 演讲标题 | 必填 |
| 受众 | 听众背景 | 技术团队 |
| 幻灯片数 | 总页数估算 | 15 张 |
| 大纲 | 章节/要点列表 | 必填（可与主题合并描述） |
| 讲者姓名 | 封面署名 | 用户名 |
| 讲者身份 | 封面副标题 | 可选 |
| 输出文件名 | 生成的 HTML 文件名 | `[主题英文短名].html` |
| 输出目录 | 写入位置 | 当前目录 |

## 工作流程

### Step 1：读取框架

框架源文件 `assets/skills-deck.html`（skill 内置）分三段，只使用前段和后段：

| 段 | 行号 | 内容 | 操作 |
|----|------|------|------|
| **前段** | 1–4812 | head / 全部 CSS / body 开头 / deck 容器 / 品牌 logo 块 | 完整读取 |
| **原内容** | 4813–7333 | 框架自带的演示幻灯片 | **跳过，不使用** |
| **后段** | 7334–8468 | 导航栏 / 进度条 / 备注面板 / Prism · jsPDF · html2canvas / 主脚本 / 关闭标签 | 完整读取 |

```
用 Read 工具读取 skill 内置 assets/skills-deck.html：
  offset=0, limit=4812   → 存为 PRE_CONTENT
  offset=7333, limit=1135 → 存为 POST_CONTENT
```

> 后段包含 PDF 导出按钮和全部交互 JS，生成文件**必须包含**它，否则翻页/PDF/备注全部失效。

### Step 2：读取幻灯片模板

```
读取 slide-patterns.md（与本 SKILL.md 同目录）
```

这是所有幻灯片的 HTML 模板库，包含封面、章节幕、内容、代码、流程图、对比、结束、金句等 8 种模式，后续生成时按模式填充。

### Step 3：规划幻灯片结构

根据用户大纲，确定幻灯片列表：

```
封面（cover）× 1
讲者介绍（可选）× 1
章节幕（chapter）× N（按大纲章节数）
内容页（content / code / mermaid / compare / quote）× M（每章 2–4 页）
结束页（end）× 1
```

每张幻灯片确定：
- `id`：`s01`、`s02`… 顺序递增
- `type`：封面选哪种、内容选哪种
- `title` / `core` / `points`：内容摘要
- `notes`：演讲者口播（自然语言，100–300 字）

### Step 4：生成幻灯片 HTML

按 `slide-patterns.md` 中的模式逐页生成 `<section>` 块：

- 每页 `id` 唯一，从 `s01` 顺序递增
- `data-notes` 写入口播内容（单行，引号用 `&quot;` 转义）
- 内容密度：要点 ≤ 5 条，每条 ≤ 20 字，核心句 ≤ 30 字
- 代码页用 Prism，流程图用 Mermaid（见 slide-patterns.md）
- 章节幕不加 `.slide-core`，只用 `.ch-layout`
- 封面**无图**时使用 slide-patterns.md 的「无封面图版」（含装饰右半区）

### Step 5：组装输出文件

```
PRE_CONTENT（前段，行 1–4812）
  └─ 替换 <title>…</title> → <title>演讲标题 · 讲者姓名</title>
+ 生成的 <section> 块（所有幻灯片）
+ POST_CONTENT（后段，行 7334–8468）
  └─ 替换 pdf.save('skills-deck.pdf') → pdf.save('演讲主题slug.pdf')
```

slug 规则：演讲主题的英文或拼音短名，如 `claude-code-intro`、`ai-skills`。

### Step 6：写入文件

用 Write 工具将完整 HTML 写入用户指定路径，写入后输出：
- 文件绝对路径
- 幻灯片总数
- 快捷键说明：`← →` 翻页 · `N` 备注 · `ESC` 索引 · 点击「PDF」导出

生成前对照核验：

- [ ] 前段（行 1–4812）完整，包含所有 CSS
- [ ] 后段（行 7334–8468）完整，包含导航和 JS
- [ ] `<title>` 已替换为演讲标题
- [ ] `pdf.save(...)` 已替换为演讲主题 slug
- [ ] 封面有主题、副标题、讲者信息
- [ ] 每章有章节幕（chapter slide）
- [ ] 每页有 `data-notes` 口播内容（非空）
- [ ] 代码块使用正确的 `language-*` class
- [ ] 文件末尾有结束页
- [ ] HTML 结构完整，无未闭合标签（每个 `<section>` 对应一个 `</section>`）

## 交互能力（自动继承，无需配置）

| 能力 | 操作 |
|------|------|
| 键盘翻页 | `←` `→` 或空格 |
| 演讲者备注 | `N` 键开关 |
| 全局索引 | `ESC` 缩略图跳转 |
| 代码高亮 | Prism.js，支持 bash/yaml/js/ts/python |
| 流程图 | Mermaid.js，翻到该页时渲染 |
| 触屏翻页 | 左右滑动 |
| PDF 导出 | 点击导航栏「PDF」按钮 |
| 进度指示 | 底部页码 |

## 注意事项

- 框架前段/后段（CSS/JS）只做两处字符串替换，其余**不修改**
- 后段（7334–8468）**不可省略**，省略则生成文件无翻页、无 PDF、无备注
- 品牌 logo（`chanyanxueyuan_white.png`）保留在框架层，用户可后续替换
- Mermaid 翻到该页时自动渲染，无需手动初始化
- 图片资源如无本地文件，省略 `<img>` 标签，使用「无封面图版」封面
