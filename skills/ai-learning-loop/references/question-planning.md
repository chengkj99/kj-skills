# Question Planning and Research Scout

Use this reference when the user starts from a broad topic, has no material, or asks for resource research.

## Goal Compression

Convert "I want to learn X" into:

```text
I want to learn X so I can produce Y for audience/context Z.
```

If Y is missing, infer one practical output. Good defaults:

- 技术/AI 编程主题 -> 口播稿、教程小节、playbook
- 成长/方法论主题 -> 短文、公众号大纲、咨询框架
- 工具/产品主题 -> 入门路线、实战清单、对比判断

## No-Material Research Pack

When the user has no material, produce the smallest useful pack:

```markdown
## 学习目标
...

## 必读 3 个
- ...

## 选读 3-5 个
- ...

## 暂时跳过
- ...

## 阅读顺序
1. ...

## 第一张表达卡片任务
...

## 资料风险
- 时效风险：
- 立场风险：
- 实践验证风险：
```

## Source Selection

Prefer:

1. Official docs, primary papers, source repositories, standards, release notes.
2. High-signal practitioner writeups with concrete examples.
3. Courses or videos only when they add structure that text sources lack.

Avoid:

- SEO listicles with no original insight.
- Outdated posts for fast-moving tools.
- Repeated summaries of the same original source.
- Marketing pages presented as neutral education.

## Current Information

For current tools, APIs, laws, prices, models, library versions, or product capabilities, verify with web search or official sources when available. If network/search is unavailable, state that the pack is provisional and list exact search queries to run later.
