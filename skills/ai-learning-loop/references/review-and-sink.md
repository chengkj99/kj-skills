# Review and Sink Decisions

Use this reference after drafting an expression card.

## Review Rubric

Score quickly as high / medium / low:

| Dimension | Question |
|---|---|
| Core judgment | Can the main claim be repeated in one sentence? |
| Reader pain | Does it answer a real problem? |
| Non-obviousness | Is there a mechanism or counterintuitive point? |
| Evidence | Is there an example, source, or firsthand case? |
| Boundary | Does it say when the advice does not apply? |
| Reuse value | Can it become article/lesson/video/consulting material? |

## Feedback Format

```markdown
## 评审结论
- 最强的地方：
- 最大的问题：

## 需要修的 3 点

## 只补一张卡
| 卡片标题 | 类型 | 建议落点 | 为什么必须补 |
|---|---|---|---|

## 下一轮问题
...
```

## Sink Decision

Pick at most one mandatory card by default:

- `wiki/concepts/`: concept, distinction, principle, reusable judgment.
- `wiki/playbooks/`: repeatable workflow, checklist, template, SOP.
- `wiki/sources/`: one material needs a traceable summary.
- `wiki/analysis/`: one heavy material must be deeply read.
- `wiki/synthesis/`: multiple sources must be reconciled.
- `raw/studio/funnel/formatted/`: user asked for a publishable draft.
- `raw/studio/funnel/transcripts/`: published short-video transcript.

When in a wiki repo, update index/log only if actually writing files and follow the repo's own AGENTS/CLAUDE/SCHEMA rules.
