# Expression Card Builder

Use this reference to turn a learning question and minimal input into one output card.

## Principle

One expression card should carry one core judgment. If the draft needs three unrelated points, split it into separate cards.

## Default Template

```markdown
# 标题

## 今日问题
我今天想回答的问题是：

## 核心判断
一句话说清楚这条内容的观点：

## 适用场景
这条内容回应什么读者痛点 / 使用场景：

## 正文
...

## 我还没讲透的地方
- 

## 可复用资产
- 可拆成公众号段落：
- 可升级成星球正课：
- 可延展成下一集：
- 相关概念页：
```

## Short Video Shape

For 60-120 second口播稿:

```text
Hook: 打破一个常见误解
Judgment: 给出一句核心判断
Mechanism: 解释为什么
Example: 给一个真实场景
Boundary: 说明什么时候不适用
Close: 给下一步动作或下一集问题
```

## Quality Rules

- Start from reader pain, not the author's desire to explain.
- Preserve a mechanism, not just a list of tips.
- Include one concrete example or counterexample whenever possible.
- Name the boundary so the card does not overclaim.
- Keep the card reusable: future article, lesson,咨询, or next video should be able to cite it.
