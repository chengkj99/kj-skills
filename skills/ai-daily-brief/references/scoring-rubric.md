# AI 日报价值评分规则

## 评分维度与权重

总分范围 `0-10`，按以下公式计算：

`score = 相关性×0.30 + 可执行性×0.25 + 新颖性×0.20 + 影响面×0.20 + 时效性×0.05`

### 相关性（0-10）

- 是否直接影响 AI 编程工作流（IDE、Agent、CLI、MCP、代码模型）。
- 与「程序员近期决策」关联越强，分越高。

### 可执行性（0-10）

- 是否能在 1-7 天内转化为行动（试用、迁移、验证、基准测试）。
- 给出明确入口（仓库、文档、发布说明）则加分。

### 新颖性（0-10）

- 是否为新发布、新能力、新基准结果。
- 仅重复观点或转述旧闻，得分低。

### 影响面（0-10）

- 影响范围（个人效率、团队协作、行业生态）。
- 头部厂商、主流基础设施或高采用率工具动态通常更高。

### 时效性（0-10）

- 是否落在目标窗口（昨日）内。
- 发布时间不清晰或显著滞后则降分。

## 优先级分级

- `P0`：`score >= 9.0`（高优先级，建议当天行动）
- `P1`：`8.0 <= score < 9.0`（重要，建议本周评估）
- `P2`：`score < 8.0`（观察项，作为趋势跟踪）

## 去重规则

- 同一事件多链接：保留官方原始链接。
- 文本语义高度重复（标题/摘要相近）：只保留最高分条目。

## 输出动作建议模板

- 发布/版本更新：给出「升级评估清单」
- 开源仓库/教程：给出「最小可运行验证」
- 基准测试/评测：给出「本地对照实验计划」
