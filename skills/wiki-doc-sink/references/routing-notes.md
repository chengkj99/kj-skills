# 沉淀落点路由（个人 Wiki）

> 本文件供 `wiki-doc-sink` 执行前 **Read** 加载。本 skill **仅覆盖个人** `<WIKI_ROOT>`，不处理组织/公司业务专用 Wiki。

## 三个常见落点

| 落点 | 适用内容 |
|------|----------|
| **当前项目** `docs/`、`README` 等 | 本仓库操作说明、脚本、部署、项目内案例 |
| **个人** `<WIKI_ROOT>` | 跨项目可复用的工程/运维/学习方法、个人学习与 AI 协作纪要 |
| **组织/公司业务 Wiki** | 不在本 skill 范围内；请使用你本机已配置的业务 Wiki 规则（例如 `~/.claude/rules/` 中的 bridge 与路由文件） |

## 沉淀前确认（硬性）

当用户提到以下或同义意图，且落点**未明确**时，**先简短问一句**再写文件：

- 「沉淀到知识库 / Wiki」
- 「抽取沟通 / 归档 / 摄入」
- 「通用知识 / 可复用方法论」
- 对话中同时涉及 **项目文档** 与 **Wiki**

### 建议问法（个人 vs 项目）

> 这次要沉淀到哪里？
> 1. **个人 Wiki**：`<WIKI_ROOT>`（工程通用、个人学习、跨项目经验）
> 2. **当前项目**：例如本仓库 `docs/...`（请用户指定路径）

若用户已明确路径或库名（如「沉淀到个人 wiki」「写在当前项目 `docs/guides/`」），则不再重复询问。

## 默认路由（用户已表达「要沉淀」但未指定库时）

| 内容类型 | 建议落点 |
|----------|----------|
| 当前 Monorepo 项目操作、脚本、部署、案例 | 项目仓库 `docs/`（用户指定子路径） |
| 跨项目可复用的工程/运维/学习方法 | **个人** `<WIKI_ROOT>` |

**禁止**因图省事把明显属于**当前项目专属**的内容写入个人 Wiki，或把个人通用方法论误写入当前项目的临时说明文件（除非用户明确要求）。

## 写入后

- 更新 `<WIKI_ROOT>/wiki/index.md`、向 `<WIKI_ROOT>/wiki/log.md` **追加**记录（若该库有摄入流程）。
- 在回复中注明：**实际写入的绝对路径** 与库名（个人 Wiki），便于用户核对。

## 与组织业务 Wiki 的分工

若用户问题涉及**组织内业务域**（例如信贷前端、特定产品线术语），且你本机另有业务 Wiki 规则：

- **不要**用本 skill 默认写入 `<WIKI_ROOT>`
- 改读用户本机配置的业务 Wiki bridge / 路由规则后再查询或沉淀
